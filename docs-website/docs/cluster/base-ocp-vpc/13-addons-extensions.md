## What Are Add-ons?

`Add-ons` are extra components that extend what the cluster can do.

The base cluster gives you the platform. Add-ons give you extra capabilities.

## Common Examples

Add-ons often help with:

- monitoring
- logging
- service mesh
- storage features
- security tools

## Why They Matter

A basic cluster may run applications, but real teams usually need more than that.

For example:

- monitoring helps track health
- logging helps with troubleshooting
- storage add-ons help with persistent data

## Beginner-Friendly View

You do not need every add-on at once.

Start with the add-ons that solve real needs:

1. monitoring
2. logging
3. storage support

Then move to more advanced things like service mesh only when the platform really needs them.

## Key Takeaways

- Add-ons extend the cluster beyond the basics.
- Common add-ons support monitoring, logging, storage, and security.
- Not every cluster needs every add-on.
- It is usually best to add them in a practical, step-by-step way.
